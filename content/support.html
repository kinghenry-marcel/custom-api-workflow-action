Common issues

The workflow action fails with an error

Check the error_message output field in the workflow action history. Common causes are an incorrect API URL, expired authentication credentials, or an API endpoint that is not publicly accessible.

I can't find the Custom API Call action in the workflow editor

Make sure the app is installed and authorised under Settings → Integrations → Connected Apps. If it appears as disconnected, reinstall it using the install link from the app listing.

The app was uninstalled but my workflows still reference it

Reinstalling the app will restore the connection. Existing workflow steps are not deleted when the app is uninstalled — they are simply paused until the app is reconnected.
