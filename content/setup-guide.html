Custom API Workflow Action — Setup Guide

This guide walks you through installing the Custom API Workflow Action app and using it in your HubSpot workflows to send HTTP requests to any external API.

What this app does

The Custom API Workflow Action adds a new action to HubSpot Workflows called Custom API Call. When a workflow runs this action, it sends an HTTP request to any API endpoint you configure — with full control over the method, authentication, headers, query parameters, and request body.

Use it to:

  • Notify external systems when a deal closes, a contact is created, or a form is submitted
  • Trigger webhooks in tools like Zapier, Make, or your own backend
  • Push data to a CRM, ERP, or data warehouse
  • Call internal APIs protected by bearer tokens or API keys
  • Run any HTTP integration without writing code

The action supports all standard HTTP methods (GET, POST, PUT, PATCH, DELETE) and four authentication modes: none, bearer token, API key header, and HTTP basic auth.

Step 1 — Install the app

  1. Go to Settings → Integrations → Connected Apps in your HubSpot account, or click the install link from the app listing.
  2. You will be redirected to the King Henry authorisation page.
  3. Click Connect to authorise the app. You will be asked to grant the automation scope, which allows the app to run workflow actions on your behalf.
  4. After authorising, you will be redirected back to your HubSpot account. The app will appear in your Connected Apps list.

The app does not require any additional configuration after installation. No API keys or account credentials are needed on your end — authentication to external APIs is configured per workflow action.

Step 2 — Add the Custom API Call action to a workflow

  1. Open or create a workflow in Automation → Workflows.
  2. Click the + button to add a new action.
  3. In the action search panel, type Custom API or scroll to the Connected Apps section.
  4. Select Custom API Call.
  5. The action configuration panel will open on the right.

Step 3 — Configure the action fields

Step name (required)

A label shown on the workflow canvas to identify this step. Use something descriptive like Notify Slack or Create lead in CRM. This is only for your reference inside HubSpot — it is not sent to the API.

API URL (required)

The full HTTPS URL of the endpoint you want to call. Must start with https://. You can use HubSpot workflow tokens here to insert dynamic values (e.g. https://api.example.com/contacts/{{contact.email}}).

HTTP Method (required)

Select one of: GET, POST, PUT, PATCH, DELETE.

Authentication type (required)

Select how the request should be authenticated:

  • None — No authentication header is added.
  • Bearer token — Adds an Authorization: Bearer <token> header. A Bearer token field will appear.
  • API key (header) — Adds a custom header with your API key. Fields for the header name and value will appear (e.g. header name: X-Api-Key, value: your key).
  • Basic auth — Adds an HTTP Basic authentication header. Fields for username and password will appear.

Query parameters (optional)

A JSON object of key-value pairs to append to the URL as query string parameters. Example:

{
  "format": "json",
  "source": "hubspot"
}

Custom headers (optional)

A JSON object of additional HTTP headers to include in the request. Example:

{
  "X-Custom-Header": "my-value",
  "Content-Type": "application/json"
}

Request body (optional)

A JSON object to send as the request body. Only used for POST, PUT, and PATCH requests. You can include HubSpot workflow tokens to insert contact or deal data dynamically. Example:

{
  "email": "{{contact.email}}",
  "dealName": "{{deal.dealname}}",
  "stage": "{{deal.dealstage}}"
}

Timeout (optional)

Maximum number of seconds to wait for a response before the action fails. Defaults to 10 seconds. Maximum allowed value is 60 seconds.

Step 4 — Use output fields in subsequent steps

After the Custom API Call action runs, four output fields are available for use in later workflow steps (such as If/then branches, Set property value, or additional API calls):

  • status_code (number) — The HTTP status code returned by the API. For example: 200, 201, 400, 404, 500.
  • response_body (string) — The full response body returned by the API, as a string. If the API returns JSON, this will be the raw JSON string.
  • success (boolean) — true if the status code was in the 2xx range, false otherwise.
  • error_message (string) — If the request failed (network error, timeout, or non-2xx status), this contains a description of the error. Empty on success.

Example: Branch on success

After a Custom API Call step, add an If/then branch. Set the condition to success is equal to true to route contacts differently depending on whether the API call succeeded.

Example: Store the response

Add a Set property value action after the API call. Use the response_body output token as the value to save the API's response to a HubSpot contact or deal property for later reference.

Step 5 — Disconnect or uninstall the app

  1. Go to Settings → Integrations → Connected Apps.
  2. Find Custom API Workflow Action and click Actions → Uninstall.
  3. Confirm the uninstall. Any existing workflows using the Custom API Call action will stop executing that step after removal.

Uninstalling the app does not delete any workflow data or contact properties. Existing workflow steps that referenced the action will remain in the workflow editor but will be marked as inactive.

Troubleshooting

The action shows an error in the workflow history

  • Check the error_message output field in the workflow action history for details.
  • Verify your API URL is correct and uses HTTPS.
  • Confirm your authentication credentials (token, API key, or username/password) are still valid.
  • Check that your API endpoint is publicly accessible from the internet.

My request body tokens aren't resolving

  • Ensure you are using the correct HubSpot token format: {{contact.property_internal_name}}.
  • Tokens are only available for properties on the workflow's enrollment object (e.g. contact tokens in a contact-based workflow).

The request times out

  • The maximum timeout is 60 seconds. If your API consistently takes longer than this, it is not suitable for use with this action.
  • Consider increasing the timeout value in the action configuration.
